# 正常风格 (Normal Style)

## 风格描述

使用标准的 Conventional Commits 格式，专业规范，适合团队协作和开源项目。

## 格式规则

- 使用类型前缀：feat, fix, docs, style, refactor, test, chore, perf
- 格式：`<type>: <description>`
- 描述使用中文，简洁明了
- 首字母不大写，结尾不加句号

## 类型说明

- **feat**: 新功能
- **fix**: 修复 bug
- **docs**: 文档更新
- **style**: 代码格式调整（不影响功能）
- **refactor**: 代码重构
- **test**: 测试相关
- **chore**: 构建过程或辅助工具的变动
- **perf**: 性能优化

## 示例

```
feat: 添加用户登录功能
fix: 修复购物车计算错误
docs: 更新 API 文档
style: 统一代码缩进格式
refactor: 重构用户认证模块
test: 添加单元测试
chore: 更新依赖包版本
perf: 优化数据库查询性能
```
