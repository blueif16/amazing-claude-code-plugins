# 详细风格 (Detailed Style)

## 风格描述

多行 commit 消息，包含详细说明和影响范围，适合重要功能更新或复杂修改。

## 格式规则

```
<type>: <简短标题>

<详细说明变更内容和原因>

影响范围：<列出受影响的模块或功能>
```

## 编写要点

- 标题行：简洁概括主要变更
- 详细说明：解释为什么做这个改动，如何实现的
- 影响范围：列出受影响的模块、文件或功能
- 使用中文描述，条理清晰

## 示例

```
feat: 添加用户登录功能

实现了基于 JWT 的用户认证系统，包括登录、注册和密码重置功能。
使用 bcrypt 加密存储密码，确保用户数据安全。

影响范围：
- 用户认证模块
- API 路由
- 数据库 schema
```

```
fix: 修复购物车价格计算错误

修复了当商品有折扣时，购物车总价计算不正确的问题。
原因是折扣计算逻辑在累加前没有正确应用。

影响范围：
- 购物车计算模块
- 订单生成流程
```
