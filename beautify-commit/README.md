# Beautify Commit

美化 Git Commit 消息的中文工具，让你的提交记录更加优雅、规范或可爱！

## 功能特点

• **多种风格** → 支持正常、详细、简洁、可爱四种预设风格，还支持自定义风格
• **模板系统** → 每种预设风格都有独立的模板文件，便于维护和扩展
• **自定义风格** → 可以输入你自己的风格描述和示例，AI 会学习你的风格
• **中文优先** → 所有提示和消息都使用中文
• **记住偏好** → 首次选择风格后自动保存，无需每次重新选择
• **智能分析** → 自动分析代码变更，生成合适的 commit 消息
• **二次元风格** → 可爱风格使用精选的颜文字和可爱表达
• **推送询问** → commit 后询问是否推送到远程仓库

## 安装

将此插件复制到你的 Claude Code 插件目录：

```bash
# 复制到项目的 .claude-plugin 目录
cp -r beautify-commit /path/to/your/project/.claude-plugin/

# 或者安装到全局插件目录
cp -r beautify-commit ~/.claude/plugins/
```

## 使用方法

在你的项目中运行：

```bash
/beautify-commit
```

### 首次使用

首次运行时，会询问你喜欢的 commit 风格：

• **正常** - 标准的 Conventional Commits 格式（feat/fix/docs 等）
• **详细** - 包含详细说明和影响范围的多行 commit
• **简洁** - 简短精炼的一句话描述
• **可爱** - 使用颜文字和二次元风格的语气
• **自定义** - 输入你自己的风格描述和示例

选择后会自动保存到 `.claude/beautify-commit.local.md`，之后每次运行都会使用你选择的风格。

#### 自定义风格

如果选择「自定义」，系统会提示你输入风格描述。你可以这样描述：

```
我喜欢使用简短的英文动词开头，然后用中文描述。
格式：<英文动词> <中文描述>

示例：
- Add 用户登录功能
- Fix 购物车计算bug
- Update API文档
- Refactor 认证模块
```

AI 会学习你的风格描述和示例，生成符合你习惯的 commit 消息。

### 风格示例

#### 正常风格
```
feat: 添加用户登录功能
fix: 修复购物车计算错误

影响范围：
- 用户认证模块

```

## 配置

插件会在 `.claude/beautify-commit.local.md` 中保存你的风格偏好。

### 更改风格

如果想更改风格，可以：
1. 删除配置文件，下次运行时重新选择
2. 或直接编辑配置文件中的 `style` 字段

### 配置文件格式

#### 预设风格配置

```markdown
---
style: normal
---

# Beautify Commit 配置

当前风格：normal
```

#### 自定义风格配置

```markdown
---
style: custom
customStyle: |
  我喜欢使用简短的英文动词开头，然后用中文描述。
  格式：<英文动词> <中文描述>

  示例：
  - Add 用户登录功能
  - Fix 购物车计算bug
  - Update API文档
  - Refactor 认证模块
---

# Beautify Commit 配置

当前风格：custom（自定义）
```

## 模板系统

每种预设风格都有对应的模板文件，位于 `templates/` 目录：

• `templates/normal.md` - 正常风格模板
• `templates/detailed.md` - 详细风格模板
• `templates/concise.md` - 简洁风格模板
• `templates/cute.md` - 可爱风格模板

模板文件包含：
• 风格描述
• 格式规则
• 示例

AI 会根据模板内容生成符合该风格的 commit 消息。你也可以自定义这些模板文件来调整风格细节。

## 工作原理

1. 检查并加载用户的风格配置（包括自定义风格）
2. 根据风格类型加载对应的模板文件或自定义风格内容
3. 分析当前 git 状态和变更内容
4. 学习仓库的 commit 历史风格
5. 根据模板和变更内容生成合适的 commit 消息
6. 自动 stage 文件并创建 commit
7. 询问是否推送到远程仓库

## 安全特性

• 自动排除敏感文件（.env, credentials.json 等）
• 不会 commit 包含密钥的文件
• 所有操作都在本地执行
